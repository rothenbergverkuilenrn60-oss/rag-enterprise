<objective>
Wire **PP-StructureV3** as the primary OCR engine for scanned PDFs behind a small,
async-safe engine abstraction, and pin its runtime dependencies — without
breaking the existing Tesseract fallback path.

Purpose: OCR-01 requires PP-StructureV3 (layout + tables + reading-order) to
replace raw PaddleOCR for `is_scanned_pdf()=true` PDFs. OCR-02 requires the call
to be a process-singleton, wrapped in `asyncio.to_thread`, gated by a bounded
`asyncio.Semaphore`, and configurable via settings. This plan delivers the
engine module + dependency pin + extractor wiring; Plan 02 delivers the
Docker bake, worker pre-warm, failure-mode handling, and the end-to-end
integration test.

Output: New `services/extractor/ocr_engine.py` module, updated
`requirements.txt` with PaddleOCR pins, new settings fields, refactored
`_extract_pdf_scanned_paddleocr` that delegates to the new engine, and a unit
test file pinning singleton + concurrency + fallback contracts.

   instantiating `PPStructureV3()` triggers model download into

2. **Tesseract retention strategy (CONTEXT Q2)** → Minimum-diff wins. Keep
   `_extract_pdf_scanned_tesseract` exactly as-is; wrap it in a
   `TesseractEngine` adapter inside `ocr_engine.py`. The existing
   `_extract_pdf_scanned_paddleocr` is collapsed to a thin shim that calls
   `get_ocr_engine().extract_pdf(...)` — no duplication of the auto/paddle/tesseract
   selection logic that already lives in `_extract_pdf_enterprise`.
3. **Concurrency default (CONTEXT Q3)** → `ocr_concurrency=2`. Justification:
   `WorkerSettings.max_jobs=10` (services/ingest_worker.py:71) means up to 10
   concurrent ARQ jobs in one worker. PaddlePaddle's CPU build uses all cores
   for matmul (research §3); allowing >2 concurrent OCR calls causes thrashing
   on a typical 4–8 core host. The semaphore caps OCR-specific concurrency
   independent of the ARQ pool size, so non-OCR work continues at full
   parallelism. Operators on bigger hosts can raise `OCR_CONCURRENCY` via env.
</objective>

    ```python
    """OCR engine abstraction — Phase 7 OCR-01/02.

    Hosts the PP-StructureV3 singleton, the asyncio.to_thread + Semaphore
    plumbing, and a small Protocol so the extractor layer never imports
    paddleocr directly.
    """
    from __future__ import annotations
    import asyncio
    from functools import lru_cache
    from pathlib import Path
    from typing import Any, Protocol

    from loguru import logger
    from config.settings import settings

    # Module-level semaphore — created lazily so import-order doesn't bind it
    # to the wrong event loop. See _semaphore() below.
    _sem: asyncio.Semaphore | None = None

    def _semaphore() -> asyncio.Semaphore:
        global _sem
        if _sem is None:
            _sem = asyncio.Semaphore(settings.ocr_concurrency)
        return _sem


    class OcrEngine(Protocol):
        name: str
        async def extract_pdf(self, file_path: Path) -> dict[str, Any]: ...


    @lru_cache(maxsize=1)
    def _paddle_pipeline() -> Any:
        """Singleton PP-StructureV3. Loads ~600MB–1.2GB on first call."""
        from paddleocr import PPStructureV3   # local import: optional dep
        logger.info("[OCR] Initializing PP-StructureV3 singleton (first call)")
        return PPStructureV3(
            use_doc_orientation_classify=False,
            use_doc_unwarping=False,
            lang="ch",
        )


    class PpStructureV3Engine:
        name = "ppstructurev3"

        async def extract_pdf(self, file_path: Path) -> dict[str, Any]:
            async with _semaphore():
                return await asyncio.to_thread(self._run_sync, file_path)

        def _run_sync(self, file_path: Path) -> dict[str, Any]:
            results = _paddle_pipeline().predict(input=str(file_path))
            pages_text: list[str] = []
            tables: list[dict[str, Any]] = []
            for page_idx, page in enumerate(results):
                # Per RESEARCH.md §3 / v1.1-track-a §3 — happy-path field access.
                page_text = ""
                try:
                    page_text = (page.markdown or {}).get("markdown_texts", "") or ""
                except AttributeError:
                    # Defensive: some pages may not have .markdown attr
                    page_text = ""
                if page_text.strip():
                    pages_text.append(f"[第{page_idx+1}页·OCR]\n{page_text}")
                # Tables — preserve pred_html for downstream
                try:
                    table_list = page.json["res"].get("table_res_list", []) or []
                    for t in table_list:
                        if t.get("pred_html"):
                            tables.append({
                                "page": page_idx + 1,
                                "html": t["pred_html"],
                            })
                except (KeyError, AttributeError, TypeError):
                    pass
            return {
                "body_text": "\n\n".join(pages_text),
                "tables": tables,
                "pages": len(pages_text),
                "title": file_path.stem,
                "engine": self.name,
            }


    class TesseractEngine:
        name = "tesseract"

        async def extract_pdf(self, file_path: Path) -> dict[str, Any]:
            # Tesseract path is sync today; offload to thread to keep the
            # async contract uniform across engines. Bounded by the same semaphore
            # so a Tesseract job and a Paddle job don't both saturate the host.
            from services.extractor.extractor import _extract_pdf_scanned_tesseract
            async with _semaphore():
                return await asyncio.to_thread(_extract_pdf_scanned_tesseract, file_path)


    def get_ocr_engine(engine: str | None = None) -> OcrEngine:
        """Select an OCR engine.

        Args:
            engine: override; defaults to settings.ocr_engine. Accepts
                {"auto","paddle","tesseract"}. "none" is handled by the caller
                (extractor) and never reaches this function.

        Behaviour:
            - "tesseract" → TesseractEngine
            - "paddle"    → PpStructureV3Engine (raises ImportError on missing dep)
            - "auto"      → PpStructureV3Engine if paddleocr importable, else
                            TesseractEngine with a warning (no silent regression).
        """
        eng = (engine or settings.ocr_engine or "auto").lower()
        if eng == "tesseract":
            return TesseractEngine()
        if eng == "paddle":
            return PpStructureV3Engine()
        # auto
        try:
            import paddleocr  # noqa: F401  — probe only
            return PpStructureV3Engine()
        except ImportError:
            logger.warning(
                "[OCR] PaddleOCR not installed (paddleocr import failed); "
                "falling back to Tesseract. Install paddlepaddle==3.0.0 + "
                "paddleocr[doc-parser]==3.1.* to enable PP-StructureV3."
            )
            return TesseractEngine()
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-uploaded PDF → ExtractorService | Untrusted file content is fed to PP-StructureV3 / PyMuPDF |
| Settings env vars → process config | `OCR_CONCURRENCY` and `OCR_TIMEOUT_SEC` cross from environment to the worker |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | requirements.txt pin | mitigate | Exact `==` and `==3.1.*` pins; supply-chain integrity left to project's existing pip resolution (no Phase 7 change to lockfile policy) |
