## v1 Requirements

### Track A — Image PDF Retrieval Depth

- [x] **OCR-01 (A-1)**: PP-StructureV3 layout-aware OCR for scanned PDFs; pure-image PDF ingest produces non-empty `chars > 0` and at least one chunk per page span via `/ingest` endpoint
  - *Outcome: Validated — OcrEngine module + PP-StructureV3 singleton; `data/raw/GB4785-2019.pdf` e2e integration test GREEN*

- [x] **OCR-02 (A-2)**: Async-safe OCR invocation with bounded concurrency (`settings.ocr_concurrency`) and Docker-baked models; first OCR request <5s after worker startup; tenacity retry once on `settings.ocr_timeout_sec` exceeded; failures surface in `IngestionResponse.extraction_errors`
  - *Outcome: Validated — Bounded semaphore + ARQ worker pre-warm + tenacity timeout retry + Tesseract documented fallback*

- [x] **META-01 (A-3)**: Section-heading enrichment in chunk content; structured `section_id` / `section_title` in metadata; `content_with_header` form `f"{section_id} {section_title}\n\n{body}"`; page numbers and section numeric IDs do NOT appear in embedded text
  - *Outcome: Validated — Chunker GB section walker + `_GB_HEADING_RE` + `structure_nodes_to_chunks` walker. 9/9 unit tests GREEN. D-02 enforced.*

- [x] **META-02 (A-4)**: pgvector metadata-filter retrieval with iterative scan; filtered query returns matching chunk in top-3; B-tree expression indexes on `(metadata->>'page_number')::int` and `metadata->>'section_id'`; `SET LOCAL hnsw.iterative_scan='strict_order'` + `hnsw.ef_search` GUCs; RLS predicate runs BEFORE GUCs and SELECT
  - *Outcome: Validated — `_build_filter_where` helper + filtered `search()` path. 8/8 PgVectorStore unit tests + 4/4 integration tests against live PostgreSQL + pgvector ≥ 0.8.0 GREEN. T-08-01 mitigation: filter values via asyncpg `$N` parameters.*

- [x] **QUERY-01 (A-5)**: Query-side filter extraction for `第N页` / `第N.M节` patterns; `extract_filters(query)` returns typed `FilterExtractionResult`; effective query (filters stripped) used for embedding; raw query preserved for response display
  - *Outcome: Validated — `services/nlu/filter_extractor.py` with `_PAGE_RE`/`_SECTION_RE`. 7/7 unit tests + e2e propagation integration test GREEN. Wired in both `QueryPipeline.run` and `AgentQueryPipeline.run`.*

## Out of Scope (v1.1)

The following were explicitly deferred during requirement definition:

- LLM-based filter extractor (regex-only in v1.1; LLM fallback deferred to v1.2)
- React / Vue / Streamlit frontend (single static HTML is the v1.1 ceiling)
- Reaching 80% coverage on legacy modules (deferred indefinitely; v1.1 only gates new code)
- MinerU / PP-OCRv5 raw replacement of PP-StructureV3 (research recommended PP-StructureV3 — alternatives deferred)
- OCR container isolation as a separate microservice (single-container deployment in v1.1; isolation deferred)
- Query-time pgvector `ef_search` auto-tuning (fixed value via settings in v1.1; adaptive deferred)
