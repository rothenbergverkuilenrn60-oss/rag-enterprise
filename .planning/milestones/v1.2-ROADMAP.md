## Milestone Summary

**Key Decisions:**

- Non-abstract default-raise `call_agentic_turn` on `BaseLLMClient` (not `@abstractmethod`) тАФ avoids breaking subclasses that don't need agentic mode; `NotImplementedError` is the safe fallback (Phase 11 D-01)
- `_RAW_DICT_FIELDS = {"input"}` lock in `ToolCall` model тАФ prevents Pydantic from coercing the opaque `input` dict into a typed model, preserving arbitrary tool schemas (Phase 11 D-B2)
- Wire fixtures hand-curated against real provider SDKs (not generated) тАФ ensures tests exercise actual parsing logic against real response shapes (Phase 11-02 decision)
- `parallel_tool_calls=True` explicit in OpenAI adapter; `disable_parallel_tool_use=False` explicit in Anthropic adapter тАФ defaults exist, but explicit makes the contract auditable (AGENT-02)
- `zip(turn.tool_calls, tool_outputs)` for result correlation тАФ preserves `tool_call.id` round-trip without additional bookkeeping (Phase 11-04)

**Issues Resolved:**

- OpenAI `agent_mode=True` silently fell back to `QueryPipeline` in production (the project default provider is OpenAI); this was invisible to callers тАФ fixed by removing `services/pipeline.py:599-604` gate
- `stop_reason` mapping divergence between Anthropic (`"tool_use"`) and OpenAI (`"tool_calls"`) тАФ absorbed inside each adapter's `call_agentic_turn`

**Issues Deferred:**

- True swarm with fork agents (AGENT-03) тАФ deferred to v1.3; requires multi-agent orchestration, inter-agent coordination, and stop conditions
- Live Anthropic integration test тАФ mock-tested in v1.2; live test deferred until `ANTHROPIC_API_KEY` available in CI
- Streaming SSE for agentic responses тАФ deferred to v1.3

**Technical Debt Incurred:**

- `agent_mode: bool = False` field in `utils/models.py:215` stays as the user-facing toggle but is now inert without the old Anthropic gate; the field name may mislead readers into thinking non-Anthropic paths are still gated (low severity тАФ field is documented in PR)
