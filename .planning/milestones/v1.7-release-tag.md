## Step 1 — Verify HEAD is the merge commit

```bash
git checkout master
git pull --ff-only origin master
git log --oneline -5
```

Confirm the v1.7 milestone PR's merge commit is `HEAD` (or very close to it).
If not, STOP — figure out why before proceeding.

---

## Step 2 — Construct the tag annotation

The tag annotation is the §Highlights paragraph plus a 4-line phase summary.
Construct inline (small enough to embed directly):

```bash
TAG_MSG="v1.7.0 — Memory Tech-Debt Burn-Down

Phase 26: Memory Infra Hygiene (TD-01, TD-03, TD-07)
Phase 27: Test Isolation + Memory Reliability (TD-02, TD-04, TD-05, TD-06)
Phase 28: Doc Sweep + v1.7 Release (DOC-01)

Pure refactor + reliability milestone. Zero new user-facing capabilities;
production-cleans the memory subsystem before adding more features."

echo "$TAG_MSG" | head -10
```

Verify the output shows the headline + phase bullets + thesis paragraph.

---

## Step 3 — Cut + push the annotated tag

```bash
git tag -a v1.7.0 master -m "$TAG_MSG"
git tag -v v1.7.0 | head -5
git push origin v1.7.0
```

Verify on GitHub: `https://github.com/rothenbergverkuilenrn60-oss/rag-enterprise/releases/tag/v1.7.0`
shows the tag with the annotation text visible.

---

## Step 4 — Publish the GitHub release

```bash
gh release create v1.7.0 \
    --title "v1.7.0 — Memory Tech-Debt Burn-Down" \
    --notes-file docs/release-notes-v1.7.md \
    --verify-tag
```

`--verify-tag` confirms the tag was pushed before creating the release.
`--notes-file docs/release-notes-v1.7.md` attaches the public 5-section
release notes as the release body.

# Edit .planning/STATE.md frontmatter:

---

## Rollback (if Step 5 reveals a problem)

```bash
# Delete remote tag:
git push --delete origin v1.7.0
# Delete local tag:
git tag -d v1.7.0
# Delete GitHub release:
gh release delete v1.7.0 --yes
```

Fix the underlying issue on `master`, then re-run Steps 2–5.
