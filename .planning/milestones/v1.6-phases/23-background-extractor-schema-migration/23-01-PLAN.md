| T-23-01-T1 | Tampering | f-string DDL with `settings.embedding_dim` | mitigate | `settings.embedding_dim: int = 1024` — typed as int by Pydantic BaseSettings, NOT user-controlled. f-string interpolation safe because the value is a typed integer, not a string. No bind parameters possible for DDL (PostgreSQL constraint). |
| T-23-01-T2 | Tampering | pgvector codec registration | mitigate | `register_vector` registers a typed codec for the `vector` PG type; subsequent `INSERT ... $N::vector` binds use asyncpg's native typed-parameter API (Plan 02), not string interpolation. Documented in RESEARCH.md §Pitfall #4. |
