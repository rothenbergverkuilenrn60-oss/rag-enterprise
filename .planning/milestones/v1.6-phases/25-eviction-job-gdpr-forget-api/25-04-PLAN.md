<objective>
Implement the `DELETE /api/v1/memory/forget` FastAPI endpoint in a new `controllers/memory.py` module and wire the router into the application (Wave 2 — depends on Plans 25-01 and 25-02).

The endpoint shape derives from 25-PATTERNS.md Analog 2 with four eng-review amendments. Key invariants:
- `Depends(get_current_user)` declared BEFORE `Header(alias="X-Confirm-Delete")` in function signature (Pitfall 7 — auth dep must come first)
- `Header(default=None, alias="X-Confirm-Delete")` (not `Header(...)`) — manual 400 check, not FastAPI 422 (Pitfall 6)
- **T9**: body checks role 403 FIRST, then header 400 (fail-closed on identity; eliminates 4xx-ordering info leak that lets attackers probe endpoint existence by varying header)
- `target_tenant_id = user.tenant_id` from JWT — never from a query param (D-1.2). **T3** documents that an admin from tenant A calling forget for a user whose facts live in tenant B receives 200/deleted=0 (idempotent no-op; admin is JWT-scoped)
- Audit row written AFTER `mem.forget_user()` returns, with actual `deleted_row_count` (D-2.3 / SP-6). **T1**: the audit_svc.log() call is wrapped in try/except. If it raises, the would-be detail dict is logged at ERROR level with operation='forget_audit_log' + payload, and the endpoint returns 200 anyway. GDPR action already committed; audit failure must not propagate to client (would surface as confusing 500 on the success path).

**T2** mount: `controllers/memory.py` defines `router = APIRouter(prefix=settings.api_prefix)` mirroring `controllers/api.py:44`. `main.py` near line 386 adds `from controllers.memory import router as memory_router` + `app.include_router(memory_router)` mirroring the existing `app.include_router(router)` line for controllers/api.py.
