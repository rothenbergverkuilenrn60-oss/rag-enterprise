<read_first>

- services/agent/__init__.py (Planner, Executor, get_planner, get_executor public API)

1. Rewrite `AgentQueryPipeline.run` body. Suggested structure (~30-50 lines):
   ```python
   async def run(self, req: GenerationRequest) -> GenerationResponse:
       audit_start = time.perf_counter()
       planner = get_planner()
       executor = get_executor()
       messages = self._build_initial_messages(req)
       parallel_factor_log: list[int] = []
       for iteration in range(MAX_ITERATIONS):  # MAX_ITERATIONS = 5, module-level constant
           plan = await planner.plan_from_messages(messages)
           parallel_factor_log.extend(len(g) for g in plan.parallel_groups)
           tool_results = await executor.execute_plan(plan, req=req)
           messages = self._append_tool_results(messages, plan, tool_results)
           # final composing call: feeds tool results back to LLM
           final_turn = await self._llm.call_agentic_turn(messages, tools=...)
           if not final_turn.tool_calls:
               # final assistant turn IS the synthesizer (CONTEXT.md D-10)
               return self._build_response(final_turn, req, audit_start, parallel_factor_log)
           # if more tool_calls returned, append assistant turn and continue outer loop
           messages = self._append_assistant_turn(messages, final_turn)
       # max_iterations exhausted: return whatever we have
       return self._build_response(final_turn, req, audit_start, parallel_factor_log, exhausted=True)
   ```
2. Adapt the `Planner` API if needed: in 16-02, Planner.plan(req) takes a request. For the outer-loop iteration here we need `Planner.plan_from_messages(messages)` — add this method to Planner in this task (or refactor 16-02's Planner to accept either; pick the cleanest API). Keep parity tests passing.
3. Helpers `_build_initial_messages`, `_append_tool_results`, `_append_assistant_turn`, `_build_response` are extracted from the OLD run body — they're mechanical extracts of message-shaping logic that already existed. Audit field threading (`parallel_factor`, latency, sources_count, query_len, intent) goes into `_build_response`.
4. Run `pytest tests/unit/test_agent_pipeline_refactor.py tests/unit/test_agent_parity.py -q`. ALL must pass.
