## Phase Details

### Phase 12: Fork-Agent Swarm
**Goal:** Users issuing multi-dimension queries with `agent_mode=True` receive answers that fully address every sub-question, produced by N independent sub-agents running concurrently with isolated context.
**Requirements:** AGENT-03 (E-3)
**Depends on:** Phase 11 (v1.2 `call_agentic_turn` abstraction)
**Success Criteria:**
1. Multi-dimension query (N ≥ 2 sub-questions) dispatches N sub-agents, each holding its own `messages` list and `max_iterations` budget; no shared message history.
2. All N sub-agents execute concurrently; total latency bounded by slowest sub-agent.
3. Coordinator emits a final synthesized answer referencing all N sub-agent results.
4. `MAX_SWARM_AGENTS` (default 5) caps decomposition; `MAX_SWARM_TURNS_PER_AGENT` (default 5) stops sub-agents cleanly.
5. Audit log per swarm invocation records N, per-agent turn count, per-agent tool calls, swarm latency, synthesis latency.

**Plans:**
- [x] 12-01 (Wave 1) — Data-model foundations (`SwarmEvent` audit dataclass, `MAX_SWARM_*` settings, dispatch routing config). ✅ complete 2026-05-09
- [x] 12-02 (Wave 2) — `SwarmQueryPipeline` core (`_decompose`, `_run_sub_agent`, `_synthesize`, `asyncio.gather` parallel burst). ✅ complete 2026-05-09
- [x] 12-03 (Wave 3) — Routing + tests (`controllers/api.py` swarm > agent > default precedence; 19 unit tests). ✅ complete 2026-05-09

**Verification:** PASS 7/7 ACs ([12-VERIFICATION.md](../phases/12-fork-agent-swarm/12-VERIFICATION.md))

### Phase 13: LLM Filter Fallback
**Goal:** Users whose queries use natural language section references (not covered by regex) get correctly filtered retrieval, with zero extra cost on queries the regex already handles.
**Requirements:** NLU-02
**Depends on:** Phase 8 (regex filter extractor baseline, QUERY-01)
**Success Criteria:**
1. Regex-hit query returns `QueryFilter` without LLM call.
2. Regex-miss with natural-language section reference returns correct filter via LLM fallback.
3. Invalid LLM JSON / API error caught silently → caller receives `None`, no exception propagation.
4. Identical unmatched queries hit cache; LLM called once per cache TTL window.
5. Returned filter carries `fallback_source` field (`"regex"` | `"llm"` | `None`).

**Plans:**
- [x] 13-01 (Wave 1) — Add `FilterExtractor` class, `ExtractionResult` dataclass, `_FILTER_EXTRACT_SYSTEM` prompt, `get_filter_extractor()` singleton. ✅ complete 2026-05-09
- [x] 13-02 (Wave 2) — Migrate 4 callsites in `services/pipeline.py` from sync `extract_filters` to `await get_filter_extractor().extract`. ✅ complete 2026-05-09
- [x] 13-03 (Wave 2 parallel) — 6 new unit tests for FilterExtractor + live-LLM integration test. ✅ complete 2026-05-09

**Verification:** PASS 5/5 must-haves ([13-VERIFICATION.md](../phases/13-llm-filter-fallback/13-VERIFICATION.md))

### Phase 14: Frontend Split and DOM Modernization
**Goal:** Developers can edit JavaScript and CSS in separate files without opening a monolithic HTML file; the running UI is visually identical to the pre-split version.
**Requirements:** UI-02
**Depends on:** Phase 9 (StaticFiles mount + ui.html baseline, UI-01)
**Success Criteria:**
1. `static/ui.html` contains no inline `<script>` or `<style>` blocks; references `ui.js` + `ui.css` only.
2. FastAPI serves `ui.js` + `ui.css` alongside `ui.html` with no config changes; symlink + StaticFiles mount unchanged.
3. No inline event handlers (`onclick=`, `onsubmit=`); all wiring via `addEventListener` in `ui.js`.
4. UI renders identically to pre-split version for upload, query submission, result display flows.

**Plans:**
- [x] 14-01 (Wave 1) — Extract inline `<style>`/`<script>` from `static/ui.html` into `static/ui.css` + `static/ui.js` (IIFE-wrapped, addEventListener wiring); 4 new unit tests. main.py + symlink unchanged. ✅ complete 2026-05-09

### Phase 15: Coverage Combine and 70% Floor
**Goal:** The CI coverage report accurately reflects all exercised paths (unit + integration combined), and the global floor blocks merges that drop below 70%.
**Requirements:** TEST-04, TEST-06
**Depends on:** Phase 10 (diff-cover gate, TEST-03); Phases 12, 13, 14 (new code must be covered)
**Success Criteria:**
1. CI produces `.coverage.unit` + `.coverage.integration` separately and combines before reporting.
2. `coverage report --fail-under=70` on combined artifact passes; PR dropping below 70% blocked.
3. Per-file diff-cover gate (TEST-03, ≥ 80% on touched files) runs independently on combined `.coverage`.
4. Service modules below 70% at v1.2 close get new unit tests covering primary execution paths.
5. CI artifacts include per-module coverage breakdown making remaining gaps immediately visible.

**Plans:**
