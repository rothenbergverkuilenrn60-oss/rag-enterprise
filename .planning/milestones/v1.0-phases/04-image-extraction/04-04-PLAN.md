<objective>
Close IMG-03 gap: `_to_retrieved_chunk()` in `services/retriever/retriever.py` reconstructs `ChunkMetadata` from JSONB metadata but omits `chunk_type` and `image_b64`. Every retrieved image chunk silently reverts to `chunk_type="text"` and `image_b64=""`. This plan adds the two missing `.get()` calls and a unit test that pins the correct behaviour.

Purpose: IMG-03 requires image chunks to be retrievable with their discriminator and base64 bytes intact. The vector store writes both fields correctly; the retriever must read them back.
Output: Patched `retriever.py` + expanded `test_retriever.py`.
</objective>
