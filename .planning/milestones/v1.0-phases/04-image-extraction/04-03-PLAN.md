```python
async def _chunk_images(
    self,
    images: list,          # list[ExtractedImage] — use Any for mypy if needed
    content: "ExtractedContent",
    doc_id: str,
    llm_client: object,
    start_index: int = 0,
) -> list["DocumentChunk"]:
    """
    Caption each ExtractedImage via LLM and produce DocumentChunk objects.
    D-03: catch (openai.APIError, httpx.HTTPError, anthropic.APIError) only.
    D-04: on failure, append to content.extraction_errors, continue; do not raise.
    """
    from utils.models import ExtractedImage, DocumentChunk, ChunkMetadata

    if llm_client is None:
        logger.warning(f"[Chunker] no llm_client — skipping image captioning: doc_id={doc_id}")
        return []

    chunks: list[DocumentChunk] = []

    for img_offset, img in enumerate(images):
        chunk_index = start_index + img_offset
        image_b64 = base64.b64encode(img.raw_bytes).decode()
        media_type = f"image/{img.ext}" if img.ext != "jpg" else "image/jpeg"

        try:
            caption: str = await llm_client.chat_with_vision(
                image_b64=image_b64,
                query="请描述这张图片的内容。",
                media_type=media_type,
                system=_IMAGE_CAPTION_SYSTEM,
            )
        except (openai.APIError, httpx.HTTPError, anthropic.APIError) as exc:
            logger.warning(
                f"[Chunker] image caption failed: doc_id={doc_id} "
                f"page={img.page_number}",
                exc_info=exc,
            )
            content.extraction_errors.append(
                f"Image p{img.page_number} skipped: {type(exc).__name__}"
            )
            continue

        if not caption.strip():
            logger.warning(
                f"[Chunker] empty caption: doc_id={doc_id} page={img.page_number} — skipping"
            )
            content.extraction_errors.append(
                f"Image p{img.page_number} skipped: empty caption"
            )
            continue

        chunk_id = _make_chunk_id(doc_id, chunk_index, caption)
        meta = ChunkMetadata(
            source=content.title or doc_id,
            doc_id=doc_id,
            chunk_type="image",
            image_b64=image_b64,
            page_number=img.page_number,
            chunk_index=chunk_index,
            doc_type=content.doc_type,
            language=content.language,
            chunk_level="child",
            node_type="image",
        )
        chunks.append(DocumentChunk(
            chunk_id=chunk_id,
            doc_id=doc_id,
            content=caption,
            content_with_header=caption,
            metadata=meta,
            token_count=len(caption.split()),
        ))

    logger.info(
        f"[Chunker] image chunks produced: doc_id={doc_id} "
        f"total={len(chunks)} skipped={len(images) - len(chunks)}"
    )
    return chunks
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ExtractedImage.raw_bytes → base64 → LLM API | Arbitrary bytes sent externally; must be bounded in size |
| LLM caption response → chunk content → vector store | LLM output is untrusted text stored in DB |
| extraction_errors → IngestionResponse → API caller | Error strings must not leak internal paths or secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Denial of Service | _chunk_images captioning loop — N LLM calls per doc | mitigate | capped at 50 images by Plan 02 extractor; each call is independent; no batching needed |
| T-04-03-02 | Tampering | LLM caption injection — adversarial content in image causes prompt injection | accept | caption is stored as chunk content, not executed; no downstream eval or code execution of caption text |
| T-04-03-03 | Information Disclosure | extraction_errors surface in IngestionResponse | mitigate | error strings use f"Image p{N} skipped: {type(exc).__name__}" — no stack traces, paths, or API keys exposed |
| T-04-03-04 | Elevation of Privilege | DocType.IMAGE bypass quality gate | mitigate | quality gate explicitly guarded by `if extracted.body_text:` — image-only docs have no text to validate; PII detector runs on body_text only, which is empty for image files (no PII bypass risk) |
| T-04-03-05 | Denial of Service | base64 encoding large images in chunker | mitigate | Plan 02 enforces 1024×1024 thumbnail resize (D-05) before raw_bytes reaches chunker; max ~300 KB per image after resize |
</threat_model>
