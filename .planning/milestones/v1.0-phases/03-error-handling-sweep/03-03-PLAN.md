## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client ↔ API (controllers/api.py) | Untrusted request data enters; route handler exceptions must not leak internal exception text to client via HTTPException.detail |
| MCP client ↔ MCP server (services/mcp_server.py) | JSON-RPC boundary; error responses must not leak server-side exception text or internal paths |
| Process lifecycle ↔ stdout/logs (main.py startup/shutdown) | Warmup failures must log to structured logger at WARNING level to distinguish from hard errors; shutdown failures are intentionally silent per D-06 |
| Service boundary ↔ persistence/cache layers | Internal services' exception propagation must not leak DB error detail into upper-layer HTTP/MCP responses |
