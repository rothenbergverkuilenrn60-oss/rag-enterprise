<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → FastAPI endpoint | user_id is a query param — not validated as UUID format (accepted as string; 404 only on empty string per D-1.3); JWT is the auth boundary |
| JWT ↔ tenant resolution | `target_tenant_id` resolved from `user.tenant_id` — attacker cannot specify a different tenant via query param |
| Endpoint → audit_service | Audit write is post-fact; if audit write fails, the DELETE already committed (D-2.3 accepted trade-off) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-04-S1 | Spoofing | Fake JWT for admin claim | mitigate | `get_current_user` verifies JWT signature via `OIDCAuthService.verify_token`; Plan 25-04 unit tests mock this at consumer path; integration tests use real JWT |
| T-25-04-T1 | Tampering | `user_id` query param injected to delete another user's data | mitigate | Auth gate: `user.is_admin or user.user_id == user_id`; tenant from JWT only (D-1.2) |
| T-25-04-I1 | Information Disclosure | `MemoryForgetError` detail leaks internal DB message | mitigate | Controller catches `MemoryForgetError` → raises `HTTPException(500, "Memory forget failed")` — sanitized string per D-1.5 |
| T-25-04-D1 | Denial | Repeated delete calls (idempotent — 200+count=0) | accept | Idempotent by design (D-1.3); no rate-limiting in Phase 25 (v1.7+) |
| T-25-04-E1 | Elevation of Privilege | Non-admin forgets another user's data | mitigate | 403 gate: `if not (user.is_admin or user.user_id == user_id)` (D-1.1); tested in Test 3 |
| T-25-04-P1 | Privacy | Forget call not audit-logged on error path | accept | If `forget_user` raises `MemoryForgetError`, no audit row is written (D-2.3: audit-after-success only); DELETE failed so no data was actually lost; audit infra logs the 500 |
| T-25-04-SC | Tampering | npm/pip/cargo installs | n/a | Zero new packages |
</threat_model>
