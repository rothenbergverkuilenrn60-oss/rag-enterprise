    Verbatim spec:

    ```python
    """asyncpg DSN normalization helper.

    Centralizes the SQLAlchemy-style DSN transformations that asyncpg's URL parser
    does not understand natively. Currently consumed by services/memory and
    services/audit pool builders (see Plans 26-03, 26-04).

    Pure function — no asyncpg dependency, no logging, no settings import.
    """

    from __future__ import annotations


    def prepare_dsn(dsn: str) -> tuple[str, dict[str, str]]:
        """Normalize a SQLAlchemy-flavored DSN for direct asyncpg consumption.

        Two transformations, both verbatim ports of the v1.6 inline blocks they
        replace (services/memory/memory_service.py:163-172 +
        services/audit/audit_service.py:261-270):

        1. Strip `postgresql+asyncpg://` → `postgresql://` so asyncpg parses
           the scheme.
        2. Strip `?ssl=disable` / `&ssl=disable` tokens out of the URL and
           return them as a separate `{"ssl": "disable"}` kwarg dict — asyncpg's
           URL parser otherwise treats `ssl` as a server_setting and raises
           `CantChangeRuntimeParamError`.

        Returns `(clean_dsn, ssl_kwarg)`. Caller forwards `ssl_kwarg` via
        `**ssl_kwarg` to `asyncpg.create_pool` or `asyncpg.connect`.

        When no `ssl=disable` token is present, the second tuple member is an
        empty dict (not None) so callers can unconditionally splat it.
        """
        # Scheme strip — order matters: longer prefix first so `postgresql+asyncpg://`
        # doesn't get short-stripped to `postgres+asyncpg://` mid-replace.
        clean_dsn = dsn.replace("postgresql+asyncpg://", "postgresql://")
        clean_dsn = clean_dsn.replace("postgres+asyncpg://", "postgres://")  # A1 (eng-review)

        ssl_kwarg: dict[str, str] = {}
        # C1 (eng-review): order matters. Try `&ssl=disable` FIRST — strips cleanly
        # leaving `?other=1` intact. Then `?ssl=disable&` — strips the prefix-with-
        # delimiter form, leaving `other=1` intact (which we then need to re-prefix).
        # Finally `?ssl=disable` — sole query param case (no other params).
        if "&ssl=disable" in clean_dsn:
            clean_dsn = clean_dsn.replace("&ssl=disable", "")
            ssl_kwarg["ssl"] = "disable"
        if "?ssl=disable&" in clean_dsn:
            clean_dsn = clean_dsn.replace("?ssl=disable&", "?")
            ssl_kwarg["ssl"] = "disable"
        if "?ssl=disable" in clean_dsn:
            clean_dsn = clean_dsn.replace("?ssl=disable", "")
            ssl_kwarg["ssl"] = "disable"

        return clean_dsn, ssl_kwarg
    ```

    No asyncpg import. No logging. No settings. No type stubs needed (returns native tuple of `str` and `dict[str, str]`).

    **C1 fix discipline:** the ordering above handles all four ssl token positions correctly:
    - `?ssl=disable` (sole param) → `""` → `postgresql://u@h/d`
    - `?ssl=disable&other=1` (first of N) → `?other=1` → `postgresql://u@h/d?other=1`
    - `?other=1&ssl=disable` (last of N) → `?other=1` → `postgresql://u@h/d?other=1`
    - `?a=1&ssl=disable&b=2` (middle of N) → `?a=1&b=2` → `postgresql://u@h/d?a=1&b=2`

    Add a unit test for the middle-of-N case if you want bullet-proof coverage (optional — same code path as the &-prefix branch).
