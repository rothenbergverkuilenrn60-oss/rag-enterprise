<objective>
Fix the +14 event-loop singleton leak sites in `tests/integration/` newly exposed by the Phase 27-02 `@pytest.mark.uses_redis` marker rollout. Default each site to the `create_app()` factory pattern (Phase 27 TD-02 evidence); sites that don't fit factory get an explicit per-test `event_loop` fixture. Grow `_SINGLETON_INVENTORY` accordingly. **Enumeration runs on a PG-enabled host at Task 0** — this plan halts and surfaces a resume command if PG is missing.

Purpose: Stale module-level singletons constructed under one event loop survive into the next test's loop and trip `RuntimeError: There is no current event loop in thread 'MainThread'`. Same root cause as Phase 27 TD-02; this plan extends the fix to the +14 newly-exposed sites.

Output: 14 (± enumeration delta) test files remediated; `_SINGLETON_INVENTORY` updated; integration suite green; remediation choice (factory vs per-test fixture) recorded per site in SUMMARY.
</objective>
