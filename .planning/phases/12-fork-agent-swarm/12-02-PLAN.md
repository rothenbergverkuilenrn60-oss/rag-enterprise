    ```python
    async def _decompose(self, query: str) -> list[str]:
        """Decompose a multi-dimension query into independent sub-questions (D-02).

        Returns single-element list `[query]` when:
          - LLM output cannot be parsed as JSON
          - Parsed result is not a list
          - Parsed list is empty after strip+dedup
        """
        raw: str = await self._llm.chat(
            system=_COORDINATOR_SYSTEM,
            user=query,
            temperature=0.0,
            task_type="generate",   # main model — see Pitfall 4
        )

        # Extract first JSON array substring (LLM may wrap in prose despite instruction).
        match = re.search(r"\[.*\]", raw, re.DOTALL)
        if match is None:
            logger.warning(f"[Swarm] coordinator returned no JSON array; falling back to N=1. raw={raw[:200]!r}")
            return [query]

        try:
            parsed = json.loads(match.group(0))
        except (json.JSONDecodeError, TypeError) as exc:
            logger.warning(f"[Swarm] coordinator JSON parse failed: {exc!r}; falling back to N=1. raw={match.group(0)[:200]!r}")
            return [query]

        if not isinstance(parsed, list):
            logger.warning(f"[Swarm] coordinator returned non-list ({type(parsed).__name__}); falling back to N=1.")
            return [query]

        # Strip + dedup while preserving order; drop empty/non-string entries.
        seen: set[str] = set()
        sub_questions: list[str] = []
        for item in parsed:
            if not isinstance(item, str):
                continue
            stripped = item.strip()
            if not stripped or stripped in seen:
                continue
            seen.add(stripped)
            sub_questions.append(stripped)

        if not sub_questions:
            logger.warning("[Swarm] coordinator returned empty list after cleanup; falling back to N=1.")
            return [query]

        # Cap at MAX_SWARM_AGENTS (D-09).
        return sub_questions[: self.MAX_SWARM_AGENTS]
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP body → `GenerationRequest.query` (then to coordinator LLM) | Untrusted user input crosses here |
| Coordinator LLM output → `_decompose` JSON parser | LLM output is untrusted (prompt injection / hallucinated JSON / non-JSON prose) |
| Sub-agent tool-call args → `_execute_tool_call` | LLM-generated structured args; flow through tenant filter `tf` |
| Sub-agent loop state → `messages` list per coroutine | Cross-coroutine boundary — must NOT share reference (Pitfall 1) |
| Pipeline → `AuditService.log(AuditEvent)` | Internal trust boundary — detail dict shape must remain backward compatible with downstream audit consumers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-02-01 | Tampering | Coordinator LLM output crafted (via prompt injection in user query) to return many sub-questions or non-JSON | mitigate | `_decompose` caps result at `MAX_SWARM_AGENTS=5` (D-09) regardless of LLM output; non-list / non-array / parse failure → fall back to `[query]` (single-agent path); `re.search(r"\[.*\]")` extracts only the first JSON array; `json.loads` on extracted substring (no `eval` — Pitfall 6); empty/non-string entries dropped |
| T-12-02-02 | Denial of Service | Sub-agent runaway loop (LLM keeps requesting tool calls forever) | mitigate | Hard `for iteration in range(self.MAX_SWARM_TURNS_PER_AGENT)` bound (default 5); for-else clause emits warning marker if max turns reached; tool calls inside the loop are bounded by their own retriever-level top_k cap |
| T-12-02-03 | Information Disclosure | Cross-coroutine state leak — one sub-agent's messages or tool results visible to another | mitigate | Each `_run_sub_agent` constructs `messages: list[dict[str, Any]] = [...]` as a fresh local literal (Pitfall 1); `all_chunks` and `tool_calls_count` are local to the coroutine; only the `_SubAgentResult` is returned via gather; tenant filter `tf` is applied per sub-agent (inherited from coordinator's tenant context, not user-controllable per sub-question) |
